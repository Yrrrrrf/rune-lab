# Rune Lab v0.5 Integration Overview

This document summarizes the layout refactoring, decoupling architecture, and
the integration of the browser-only `@chenglou/pretext` typesetting engine.

## 1. Decoupled Core Architecture

To guarantee that the **Layout plugin is purely layout-oriented** and completely
agnostic to visual themes, languages, or monetary currencies, all stores and
components have been migrated to their proper domain-vocabulary plugins:

- **Layout Plugin (`rune-lab.layout`)**:
  - Contains strictly the layout configuration and preset options (`page`,
    `docs`, `workspace`).
  - Implements the new pretext typeset renderer: `Text.svelte`,
    `RichText.svelte`, and responsive layout measuring.
- **Theme Plugin (`rune-lab.theme`)**:
  - Implements the theme state and DaisyUI CSS theme integration.
  - Contains `ThemeSelector.svelte` and the `theme` configuration store.
- **i18n Plugin (`rune-lab.i18n`)**:
  - Implements the `language` slot store and language selections.
  - Contains `LanguageSelector.svelte`.
- **Money Plugin (`rune-lab.money`)**:
  - Implements currency and exchange rates.

Backward-compatible re-exports and context accessor mappings are maintained in
[mod.ts](file:///home/yrrrrrf/Documents/lab/code/typescript/rune-lab/src/packages/plugins/layout/src/mod.ts)
to prevent breaks in legacy layout users.

## 2. Integrated Pretext Typesetting Engine

- Integrated the real `@chenglou/pretext` canvas typesetting library for
  high-performance responsive text measurements.
- Created
  [PretextTextMeasurer](file:///home/yrrrrrf/Documents/lab/code/typescript/rune-lab/src/packages/plugins/layout/src/text/adapter.ts)
  adapter implementing the `TextMeasurer` port defined in the `core` package.
- Solved Vitest/TypeScript compilation and type resolution issues with pretext
  exports (including subpaths like `"pretext/rich-inline"`).

  component element tags.
